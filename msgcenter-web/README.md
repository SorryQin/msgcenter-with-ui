# MsgCenter Web Console 前端

基于 HTML + CSS + Vanilla JS 构建的消息中心管理后台，无需任何构建工具，直接在浏览器打开即可使用。

## 使用方式

### 方式一：通过后端服务访问（推荐）

启动后端服务后，直接访问：

```
http://localhost:8082/console/
```

后端已配置静态资源映射和 CORS，无需额外配置。

### 方式二：独立打开 HTML 文件

直接双击 `index.html` 打开，此时需要后端服务（`localhost:8082`）正常运行，且浏览器允许跨域请求。

> 如果浏览器阻止跨域，可以安装 **Allow CORS** 插件，或使用方式一。

## 功能模块

### 📊 系统状态
- 消息队列模式、Kafka、MySQL、Redis 连接状态
- 渠道熔断状态实时展示
- 熔断配置参数显示

### 📝 模板管理
- 创建 / 编辑 / 删除消息模板
- 模板内容支持 `${占位符}` 语法
- 支持 Email / SMS / Lark 三种渠道
- 模板搜索过滤

### 🚀 发送消息
- 选择模板后自动解析占位符参数
- 填写收件人、主题、优先级
- 支持渠道覆盖（可选）
- 支持定时发送（设置时间戳）
- 发送历史实时记录

### 📋 发送记录
- 通过 msgId 查询消息发送状态
- 查看消息状态（待处理 / 成功 / 失败）
- 查看错误信息和重试次数

## 前端文件结构

```
msgcenter-web/
├── index.html        # 主页面（单页应用，多标签切换）
├── css/
│   └── styles.css    # 全部样式
└── js/
    ├── api.js        # API 请求封装
    └── app.js        # 业务逻辑
```

## 技术要点

- **无框架**：纯 HTML + CSS + Vanilla JS，零依赖
- **跨域**：后端已配置 CORSFilter，允许 `localhost:*/` 跨域访问
- **响应式**：适配桌面端，支持 1200px / 768px 断点
- **状态管理**：内存中的简单状态（模板列表、发送历史）
